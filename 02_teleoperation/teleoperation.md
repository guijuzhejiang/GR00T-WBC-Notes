# 遥操作指南

全身遥操是这套系统里最危险也最有趣的部分。SONIC 策略会实时追踪你的全身动作，包括脚部移动，所以得认真对待安全问题。

## 硬件要求

- PICO VR 头显（含手柄）
- 足部追踪器（安装在鞋子或脚踝上）
- 吊架（gantry）——首次使用必须，防止机器人摔倒
- 同一局域网（低延迟，有线优先）

## 启动流程

需要同时开三个终端：

**终端 1 — MuJoCo 仿真器**（仿真模式用，真机跳过）
```bash
source .venv_teleop/bin/activate
python gear_sonic/scripts/run_sim_loop.py
```

**终端 2 — C++ 部署程序**
```bash
cd gear_sonic_deploy

# 仿真模式
bash deploy.sh --input-type zmq_manager sim

# 真机模式
bash deploy.sh --input-type zmq_manager real
```

**终端 3 — PICO 追踪服务**
```bash
source .venv_teleop/bin/activate
python gear_sonic/scripts/pico_manager_thread_server.py --manager
```

## 操作步骤

1. 戴上 PICO 头显和手柄，确认足部追踪器牢固固定
2. 让机器人松弛地支撑在吊架上（策略启动后会自己维持平衡）
3. 站到标准校准姿势：直立、两脚并拢、双臂自然下垂
4. 按 **A+B+X+Y**：策略初始化并进入 Planner 模式
5. 按 **A+X**：切换到 Pose 模式，全身遥控正式激活
6. 正常走动即可，机器人会跟随你的动作
7. 结束时按 **A+B+X+Y**：停止策略

## 控制器按键说明

| 按键 | 功能 |
|------|------|
| A+B+X+Y | 初始化 / 停止策略 |
| A+X | 切换到全身追踪模式（Pose mode） |
| Menu 长按 | 暂停动作流，松开继续 |
| O（键盘） | 紧急停止（需要操作员在旁守候） |

## 校准姿势要点

校准做不好，整个遥操效果都会偏差：

- 站直，眼睛平视前方（不要低头看手柄）
- 两脚并拢，脚尖平行靠近
- 双臂完全悬垂在身体两侧
- 按完 A+B+X+Y 后保持这个姿势 1-2 秒

**如果感觉机器人偏移或动作不准，先 A+B+X+Y 停下，重新校准。**

## 走动技巧

自然走就行，策略是在自然人体动作数据上训练的。

**好的做法：**
- 正常步幅、正常手臂摆动
- 重心转移自信、流畅
- 先从舒适的步行速度开始，熟悉后可以加速

**常见错误：**
- 动作过慢、犹豫——反而更容易让机器人踉跄
- 试图配合机器人移动——会形成反馈死循环，直接摔跤

**进阶动作**（熟练后尝试）：
- 转向：用上身带动，自然迈步
- 伸手：平滑延伸，不要突然
- 下蹲：正常弯膝，保持重心
- 侧步：正常重心转移

## 模式切换安全意识

切换模式是最容易出危险的时机。

**危险场景：**
1. 机器人在 Planner 模式，直立站立
2. 你正在蹲着或手臂抬起
3. 直接按 A+X 切换 → 机器人剧烈运动调整姿态

**安全做法：**
1. 切换前先看看机器人或可视化界面
2. 把自己的姿势调整到接近机器人当前状态
3. 再按 A+X

**Menu 键暂停：** 暂停期间机器人定格，松开 Menu 前先把自己姿势调整回去。

## 安全操作规范

- 机器人周围保持 **3米安全距离**
- 必须有第二人在电脑旁守候，随时准备按 O 紧急停止
- 在真机上操作前，先在仿真里充分练习
- 机器人第一次使用新场景时，始终用吊架保护

## 故障排查

**机器人没有追踪我的动作：**
- 检查足部追踪器电量和固定情况
- 裤子太宽松会遮挡追踪器，换紧身裤
- 改善环境光照（避免过亮或过暗）
- 重启 PICO 上的 XRoboToolKit，重新校准

**机器人突然剧烈动作：**
- 通常是切换模式时姿势不匹配导致的
- 也可能是追踪丢失或者 WiFi 丢包
- 立刻紧急停止，重新校准后再继续

**追踪抖动/踉跄：**
- WiFi 干扰——远离其他无线设备
- IMU 漂移——重新校准机器人
